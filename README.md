# cn-hk-guide

一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为 HTML 文档的集中存放与版本管理用途。

## 项目简介

本项目旨在提供一个结构化的空间，用于存放、维护和分享一系列自包含的 HTML 页面。这些页面可能涉及信息整理、工具页面、说明文档或其他静态内容，每个页面均为独立文件，互不依赖。

## 目录结构

```
cn-hk-guide/
├── pages/           # 存放所有 HTML 页面文件
├── assets/          # 页面引用的静态资源（如样式、脚本、图片等）
├── index.html       # 仓库首页 / 页面索引
└── README.md        # 本文件
```

- `pages/` 目录下每个 `.html` 文件代表一个独立页面，文件名应简洁描述页面内容。
- `assets/` 目录用于存放页面所需的共用或专用资源文件，建议按页面或类型组织子目录。

## 页面归档说明

- 每个 HTML 页面应为完整、自包含的文档，可独立在浏览器中打开。
- 页面之间不应存在硬编码的相互依赖关系，但可通过相对路径引用 `assets/` 下的资源。
- 归档的页面内容应保持稳定，如需更新请提交 Pull Request 并注明修改理由。

## 维护说明

- 本仓库欢迎对已有页面进行内容修正、样式优化或新增独立页面。
- 新增页面时，请确保不引入外部不可控资源（如第三方 CDN），以保证页面长期可访问。
- 提交前请检查页面在主流浏览器（Chrome、Firefox、Safari）中的基本显示效果。
- 仓库维护者会不定期审核 Pull Request，并可能对页面结构或命名进行微调以保持一致性。

## 贡献方式

1. Fork 本仓库。
2. 在 `pages/` 下添加或修改 HTML 文件。
3. 如有新增资源，请放置于 `assets/` 对应子目录。
4. 提交 Pull Request，并简要说明变更内容。
